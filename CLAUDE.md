## プロジェクト概要

USCPA試験の問題演習Webアプリ。「やりながら覚える」学習スタイルに合わせ、無限に問題を解ける環境を作る。

## 設計仕様（確定）

### 背景・課題（ChatGPTで演習していた際の痛み）
- AIが生成する問題の品質が低い（消去法で解けてしまう）
- 計算問題の選択肢に正解が存在しないことがある
- 誤答時にヒントではなく正解を教えられてしまう
- ChatGPT → Gemini → Notionへの手動コピペが面倒

### データ階層構造
→変更：Chapter単位 → トピックレベルに変更（2026-02-25）
→変更：プール100問 → 16問に変更（コスト・GAS上限を考慮）
```
科目（FAR / AUD / REG / BAR / ISC / TCP）
  └ 章（ナビゲーション用）
       └ 単元（トピックレベル）  ← スキップ管理の単位
            └ 問題プール（16問を維持）
                 └ 各問題：設問・正解・ヒント・種別・難易度・正誤履歴
```
- 連番トピック（例：棚卸資産の評価方法①〜④）はまとめて1単元

### 単元ごとのSheetsデータ
| カラム | 内容 |
|---|---|
| `study_flag` | 学習済みか（true/false）。trueの単元だけ出題対象 |
| `consecutive_correct` | 現在の連続正解数（誤答でリセット） |
| `skip_until` | 出題停止の終了日（nullなら停止なし） |
| `history_json` | 直近100問の正誤ログ [1,0,1,1,0...] |

### 学習ロジック
→変更：10問連続正解・10日スキップ → 8問連続正解・10日スキップ（2026-02-25）
- 単元ごとに**8問連続正解** → その単元を**10日間スキップ**（間隔反復法）
- 誤答時：正解は教えず、ヒント（考え方・着眼点）を1つだけ提示
- 誤答後：同一単元の1問目からやり直し（問題・選択肢は変更）
- 出題は4問1セット

### 問題プール管理（スマート再生成）
→変更：週次フル再生成 → 「回答済み問題だけ再生成」に変更（コスト削減のため）
- 週次スケジュール実行（GAS）
- 正誤履歴がついた問題のみ新規生成して入れ替え
- 未回答問題はそのまま残す（コスト0）
- サボった週は再生成なし → コスト0
- 入れ替え時：vocab問題はvocabで、calc問題はcalcで再生成（比率を維持）
- 入れ替え後：問題レベルの履歴はリセット、単元レベルの100問ウィンドウは継続

### 問題への付与情報
→変更：概念タグ案を検討したが「タグ設計に時間を取られる・本質的でない」として不採用
- **種別：** vocab / calculation
- **難易度：** easy / medium / hard
- **vocab:calc比率 = 60:40**（計算問題がない単元はvocabのみ）

### 計算問題の品質担保
- **2段階検証方式：** 生成（1回目）→ 自動検証（2回目）→ NGなら再生成 → 検証済みのみ保存・出題

### 科目レベルの習熟度
- 科目の習熟度 = その科目に属する全単元の「直近100問正答率」の平均

### 技術スタック
→変更：GAS web app単体 → フロントエンドをGitHub Pagesに分離（2026-02-26）
- **フロントエンド：** HTML/CSS/JS（GitHub Pages） `https://masstack-cmd.github.io/uscpa/`
- **バックエンド：** Google Apps Script（HTTP API、`doGet` で問題取得・回答記録）
- **AI：** Claude API（vocab→Haiku、calc→Sonnet）
- **DB：** Google Sheets（問題DB・進捗管理）

#### 分離の経緯
GAS web appはiframe構造のためスマホブラウザで正しくレイアウトが機能しない（viewport幅が誤検知される・iframe高さがコンテンツ分しか確保されない）。フロントエンドをGitHub Pagesに移すことで完全なモバイル対応を実現（2026-02-26）。

### UI構成
- トップ：科目選択メニュー [FAR] [AUD] [REG] [BAR] [ISC] [TCP]
- 科目選択後：章一覧 → 単元一覧（学習フラグ・習熟度表示）→ 演習開始

### 問題ソース
- メイン：abitusテキスト（docx、購入済み）
- 補助：AICPA公式（Sample Questions・Exam Blueprints）
- ネット自動収集：対象外（著作権・技術的リスク）

### 出題タイプ（4種）
1. 読み込んだ問題をそのまま出題
2. 文章問題の選択肢を変更して出題
3. 計算問題の数字と選択肢を変更して出題
4. AIオリジナル問題

### コスト感
- vocab 1問：~$0.007（Haiku×2回）
- calc 1問：~$0.033（Sonnet×2回）
- 17単元×16問の初期生成：~$4
- 通常運用（30問/日）：月1,000〜2,000円程度

- TOKEN: `uscpa-app`（GASスクリプトプロパティに保存済み）

### Claude Codeの問題生成手順
1. 指定された単元の問題を生成（vocab: 60%、calc: 40%）
2. calc問題は内部で自己検証してから出力
3. JSON形式でPythonスクリプトを呼び出してSheetsに保存

### Pythonでの送信方法
```python

with urllib.request.urlopen(req) as res:
```
